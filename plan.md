# TC Manager - 개발 계획

## 프로젝트 목표
QA 직무에서 실제로 사용할 수 있는 테스트 케이스 관리 도구
- 테스트 케이스를 프로젝트별로 관리
- URL을 입력하면 Playwright로 자동 QA 테스트 후 바탕화면에 HTML 리포트 저장

---

## 완료된 기능 ✅

### 1단계: 기본 UI 레이아웃
- [x] Next.js 15 + TypeScript + Tailwind CSS v4 + shadcn/ui 프로젝트 세팅
- [x] 사이드바 + 메인 영역 레이아웃
- [x] 프로젝트 목록 사이드바

### 2단계: 테스트 케이스 CRUD
- [x] 프로젝트 생성 / 삭제
- [x] 테스트 케이스 등록 (제목, 설명, 사전조건, 단계, 기대결과, 우선순위)
- [x] 테스트 케이스 목록 조회
- [x] 테스트 케이스 삭제

### 3단계: 상태 관리
- [x] 테스트 케이스 상태 변경 (PENDING / PASS / FAIL)
- [x] 우선순위 설정 (LOW / MEDIUM / HIGH)
- [x] 상태별 배지(Badge) 색상 구분

### 4단계: URL 자동 테스트
- [x] /url-test 페이지 - URL 입력 후 테스트 실행
- [x] Playwright Chromium 기반 자동 검사
- [x] 바탕화면에 HTML 리포트 저장 (OneDrive 바탕화면 자동 감지)
- [x] 검사 항목:
  - HTTP 상태 코드
  - 페이지 로드 시간
  - 콘솔 에러
  - 깨진 이미지
  - 링크 검사 (최대 10개)
  - 네트워크 요청 실패
  - 접근성 (H1, lang, viewport, meta description, alt 속성)
  - 동영상 요소 감지 및 재생 가능 여부
  - 오디오 요소 감지
  - 캔버스 요소 감지 (게임/인터랙티브)
  - iframe 감지 (임베디드 콘텐츠)
  - 페이지 스크린샷

## 기술적 결정 사항

| 항목 | 결정 | 이유 |
|------|------|------|
| Playwright 실행 방식 | child_process.spawn으로 별도 프로세스 | Next.js API 라우트에서 직접 실행 시 경로 문제 |
| 브라우저 경로 | /browsers 폴더 (로컬 설치) | AppData 경로가 Next.js 서버 컨텍스트에서 인식 안 됨 |
| play() 타임아웃 | 3초 | 자동재생 정책으로 인해 무한 대기 방지 |
| DB | SQLite (dev.db) | 로컬 단독 사용 목적 |
